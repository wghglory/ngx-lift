Alerts are banners that draw the user's attention to an important message. Elements of an alert, such as icons and color, indicate the type and urgency of the message's information.

Usage

Use alerts to grab the user's attention to provide critical information needed in context.

The <clx-alert /> component provides optional inputs, including alertType (defaulting to "danger"), isSmall (defaulting to false), and the ability to handle errors via error (type of HttpErrorResponse). Additionally, you can pass content to display within the alert. It's important to note that you should choose either an error or content to pass, not both. The isAppLevel input defaults to false. If you want to display a app-level alert, set true to it.

Types

There are two types of alerts: Standard and App-level.

Standard Alerts

Use standard alerts for messages that reflect the context of where the user is in the application.

App-Level Alerts
